# Bug 记录与处理跟踪

用于统一记录 Bug 的发现、分析、修复和验证状态，便于后续排查与回溯。

## 状态说明

- `🟡 待确认`：问题已记录，但尚未完成复现确认
- `🛠️ 处理中`：已确认问题，正在分析或修复
- `🧪 待验证`：已提交修复，等待测试验证
- `✅ 已解决`：修复并验证通过
- `⚪ 已关闭`：确认无需处理（非 Bug/重复/无法复现）

- 状态：🛠️ 处理中
- 严重级别：中
- 发现时间：2026-03-05
- 解决时间：-
- 影响版本：待确认
- 解决版本：-
- 记录人：sonic
- 最后更新：2026-03-05

### 现象描述
- 归档模型时，如果存在多个配置文件同名，最终只保留一个文件，其他文件被覆盖或丢失。

### 复现步骤
1. 归档包含多个同名配置文件的模型。
2. 检查归档结果目录中的配置文件。
3. 观察到同名文件仅保留一个。

### 期望结果
- 同名文件应全部保留，且能够区分（例如自动追加后缀编号）。

### 实际结果
- 仅保留一个同名文件，其余文件丢失。

### 初步分析 / 根因
- 文件保存逻辑可能使用原始文件名直接写入，导致同名覆盖。

### 解决思路
- 保存前检测目标文件名是否已存在。
- 若存在则按规则追加后缀（如 `_1`、`_2`）后再写入。

### 修复记录
- 2026-03-05：提出修复思路“名称重复添加后缀编号”。
